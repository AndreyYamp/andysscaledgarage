# Project Context — Model Blog

## 🎯 Goal

This is a personal photo blog about scale model building (mainly cars, racing, and sports models).

The goal is NOT just to document builds, but to create a **magazine-like experience**:
- aesthetic
- calm
- visual-first
- minimal text, but meaningful

The site should feel like flipping through a **beautiful printed journal**.

---

## 🧩 Core Structure

The site has 4 main sections:

### 1. Projects (main content)

Each project is a **story**, not just a gallery.

Structure of a project:

- Introduction
  - what model it is
  - why it was chosen
  - short context

- Build stages
  - preparation
  - assembly
  - painting
  - detailing
  - final touches

Each stage:
- 2–5 photos
- short captions (not long paragraphs)

- References / Notes
  - real car references
  - historical context
  - interesting facts

- Final gallery
  - 5–10 high-quality photos
  - minimal or no text

---

### 2. Gallery

A clean visual gallery with the best photos across all projects.

No process, no explanations — only visual impact.

---

### 3. Notes

Short posts (micro-articles), for example:
- painting techniques
- mistakes and lessons
- interesting references
- small discoveries

These are NOT long articles — just focused insights.

---

### 4. About

Minimal page:
- who I am
- what this blog is about
- philosophy of the project

---

## 🎨 Design Principles

- Minimalism
- Clean layout
- Lots of whitespace
- Focus on photography
- No visual clutter

Photos are the main content, text is secondary.

---

## ✍️ Writing Style

- Short and precise
- No long essays
- Captions instead of paragraphs
- Calm, reflective tone

Bad:
> Long technical explanations

Good:
> Short notes like in a printed magazine

---

## 📸 Content Style

- High-quality photos
- Natural or soft lighting
- Consistent aesthetic
- Detail shots (macro) are important

---

## ⚙️ Technical Stack

- Hugo static site generator
- Toha theme (customized)
- Content written in Markdown

---

## 🧠 How AI should behave (important)

When generating content or code:

- Prefer simplicity over complexity
- Do not over-engineer layouts
- Do not generate excessive text
- Focus on visual presentation
- Follow the "journal" style, not "blog dump"

---

## 🚫 Anti-Patterns (avoid)

- Long blog-style articles
- Overloaded pages
- Too much text between images
- Complex navigation
- Technical-looking UI

---

## ✅ Success Criteria

The site should feel:
- calm
- эстетично (aesthetic)
- like a personal curated journal
- enjoyable to scroll slowly

---

## 🧱 Content Example (Project)

A good project page should:

- load fast
- start with a strong visual
- guide the user through the build
- end with a strong final gallery
